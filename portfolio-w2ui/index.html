  • UI component libraries
    UI component libraries zijn meestal een verzameling van User Interface componenten, zoals headers, panels, accordeons, form elementen, enz. In bepaalde gevallen zijn het zelf meerdere componeten, die samen een groter, bij elkaar horend onderdeel vormen. De implementatie vindt op verschillende manieren plaast. Sommige libraries gebruiken uitsluitens CSS-stijlen binnen de HTML-code, terwijl anderen Javascript toepassen om de componenten samen te stellen. We bekijken hier beide methoden.
  • UIkit
    Download UIkit of gebruik npm om UIkit toe te voegen aan de node-modules, als je voor die workflow kiest. Je kunt ook de complete GitHub data downloaden via een link vanaf de homepage van de UIkit website. UIkit bestaat uit twee directories, nl. een css-directory en een Javascript-directory. Deze worden aan een webpagina toegevoegd m.b.v. de onderstaande code. Tijdens de ontwikkeleing van een site gebruiken we niet de minified versie om eventuele fouten op te kunnen sporen. Om het gebruik van UIkit makkelijker te maken als je het wilt inzetten voor je eigen website ontwikkeleing, is er een extensie voor VSCode met de naam UIkit 3.0 Snippets. Deze snippet kan rechtsreeks via VScode worden geïnstalleerd.
  • Container
    Een container bestaat hier uit de ul-tag met een aantal li-tags, die de verschillende items bepalen. De container ul-tag krijgt één class mee, nl. uk-accordion. Binnen de li-tag is een a-tag geplaatst dat de kop verzorgd, waarmee het betreffende item geopend of gesloten kan worden. Deze krijgt de class uk-accordion-title. Binnen de protfolio site vervangen we de sidebar door dee code, omdat we de siebar in dit voorbeeld toch niet gebruiken, Vervolgens vervangen we de accordeon-content door wat meer tekst, die we kopieren uit de modern-webdevelopment website.
